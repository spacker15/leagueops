## Interaction Contracts

### VenueAutocompleteInput

| State                      | Visual                                                                                                                                                     | Behavior                                                  |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| Empty                      | Standard `inp` styled input with MapPin icon prefix (`text-muted`)                                                                                         | Placeholder: "Search venue or enter address"              |
| Typing (< 3 chars)         | No dropdown                                                                                                                                                | No API call — debounce inactive                           |
| Typing (>= 3 chars)        | Dropdown appears below input (`z-50`, `absolute`, `bg-[#081428] border border-[#1a2d50] rounded-xl shadow-xl`)                                             | 300ms debounce, then fetch `/api/maps/autocomplete?q=...` |
| Dropdown loading           | Small spinner or no indicator                                                                                                                              | Fetch in-flight — dropdown hidden until results arrive    |
| Dropdown with results      | 1–5 prediction items; each shows main text (12px font-black condensed, white) + secondary text (10px, muted)                                               | `onMouseDown` (not onClick) fires selection before blur   |
| Venue selected             | Dropdown closes; input shows venue name; green success chip: "Venue saved" below input (10px, `text-[#22c55e]`, `bg-[#052e14]`, rounded-full, `px-2 py-1`) | Silent save — no toast needed (chip confirms inline)      |
| Freetext (no selection)    | No chip shown; plain text value saved as `location`                                                                                                        | `venue_lat/lng/place_id` stay NULL (D-02)                 |
| API error (500 from proxy) | Input stays active; no dropdown; no chip                                                                                                                   | Fail silently — do not block form submission              |

**Blur/click timing:** Use `onBlur={() => setTimeout(() => setShowVenueDropdown(false), 200)}` on input. Use `onMouseDown` (not `onClick`) on dropdown items. (Source: RESEARCH.md Pitfall 3, EventPicker.tsx existing pattern.)

**Debounce:** 300ms. Do not fire API call for queries shorter than 3 characters.

**US restriction:** The `/api/maps/autocomplete` route must include `&components=country:us` (RESEARCH.md Pitfall 4 — this is a required fix, not in existing code).

### Sharing Tab — Draft Gate

| `active`               | Full sharing content: registration URL row, QR thumbnail row, share buttons row                                                                                                                    |

### Sharing Tab — Active State Layout

Three stacked sections within the Sharing tab, each preceded by a `sectionHdr` label:

**Section 1 — Registration Link**

- Section header: "REGISTRATION LINK" (sectionHdr class)
- Read-only input showing full URL (`inp` class + `cursor-default select-all`)
- Adjacent "COPY LINK" button (Btn variant="primary" size="sm", lucide `Copy` icon left)
- On copy: `toast.success('Link copied')`, button icon briefly switches to `Check` for 2000ms
- Primary focal point: "COPY LINK" primary Btn in Section 1 — first action above the fold.

**Section 2 — QR Code**

- Section header: "QR CODE" (sectionHdr class)
- Row: 80x80px QRCodeSVG thumbnail (white bg, black fg) + right side: event name (12px, font-black, white) + registration URL (10px, muted, truncated) + "PREVIEW" ghost button (Btn variant="ghost" size="sm")
- "PREVIEW" opens Modal with 256px QR centered in white container, event name caption below, download buttons

**Section 3 — Share**

- Section header: "SHARE" (sectionHdr class)
- Row of three ghost-style buttons (Btn variant="ghost" size="sm"):
  - `Mail` icon + "EMAIL" — opens `mailto:` link
  - `MessageSquare` icon + "TEXT" — opens `sms:` link
  - (Copy button is in Section 1 — not repeated here)

## Accessibility

| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Venue input keyboard nav | Dropdown items reachable via arrow keys; Enter selects; Escape closes                                                                               |
| Touch targets            | All buttons minimum 44px height — verify Btn size="sm" meets this; add `min-h-[44px]` if needed                                                     |
