
**Internal state (extracted from EventPicker.tsx lines 71-78):**

- `venueQuery` (string) -- the display text in the input
- `venuePredictions` (array of `{ place_id, description, main_text, secondary_text }`)
- `venueSearching` (boolean)
- `showVenueDropdown` (boolean)
- `debounceRef` (useRef for setTimeout ID)

**searchVenue function (from EventPicker.tsx lines 401-420):**

- Set venueQuery to input value
- If query.length < 3: clear predictions, hide dropdown, return
- After 300ms debounce (use debounceRef with clearTimeout/setTimeout pattern):
  - fetch `/api/maps/autocomplete?q=${encodeURIComponent(query)}`
  - set predictions from `data.predictions ?? []`
  - show dropdown
- On error: clear predictions silently (per D-02 -- freetext fallback)
- Call `onLocationChange(text)` on every keystroke so parent tracks freetext

**selectVenue function (from EventPicker.tsx lines 422-438):**

- Hide dropdown
- fetch `/api/maps/details?place_id=${encodeURIComponent(placeId)}`
- If `data.name`: set venueQuery to `data.name + (data.address ? ' -- ' + data.address : '')`
- Call `onVenueSelect({ name: data.name, address: data.address || '', lat: data.lat, lng: data.lng, place_id: placeId })`
- On error: silently fail (no toast -- component should not depend on toast)
