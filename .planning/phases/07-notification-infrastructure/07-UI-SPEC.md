## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `px-2`, `py-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`, `p-4`) |
| lg | 24px | Section padding (`p-6`, `gap-6`) |
| xl | 32px | Layout gaps (`p-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Bell icon touch target: minimum 44x44px (`p-2.5` = 10px padding on a 24px icon = 44px hit area) — accessibility requirement per ui-ux-pro-max skill
- Notification dropdown item row: 40px height (`py-2.5` with 14px font) — compact list density matches CommandCenter pattern
- Tab content padding: 12px 14px (`tab-content` CSS class) — existing pattern from globals.css, do not change

Source: CLAUDE.md conventions + ui-ux-pro-max touch-target-size rule + globals.css `.tab-content`

## Interaction Contract

### Bell Icon (TopBar)
- Position: TopBar right section, between LIVE indicator and user pill
- Icon: `Bell` from lucide-react, `size={16}`, color `#5a6e9a` (muted) at rest
- Badge: absolute-positioned dot `w-2 h-2 rounded-full bg-red` when unread count > 0 (matches existing live dot pattern)
- Unread count: `font-mono text-[9px] font-black` in `bg-navy rounded-full` pill, visible when count >= 1, max display "9+" above 9
- Click: opens `NotificationDropdown` below bell — do NOT use hover
- Active state: Bell icon color `#ffffff` when dropdown open
- Touch target: `p-2.5` wrapper = 44px minimum hit area (accessibility)
- aria-label: `"Notifications — {N} unread"` or `"Notifications — no unread"`

Source: CONTEXT.md D-04 + ui-ux-pro-max touch-target-size + aria-labels rules

### NotificationDropdown
- Width: 320px, max-height: 400px, overflow-y: auto
- Background: `#061428`, border: `1px solid #1a2d50`, `rounded-b-xl shadow-2xl` — matches TopBar dropdown
- Header row: "NOTIFICATIONS" label (`font-cond text-[10px] font-black tracking-widest text-muted uppercase`) + "Mark all read" ghost link
- Notification list items: `px-4 py-2.5`, border-bottom `1px solid #1a2d50`, unread items `bg-surface-card`, read items `bg-transparent`
- Each item: alert type pill (Pill component, appropriate variant) + 1-line summary + relative timestamp (`font-mono text-[10px] text-muted`)
- Empty state: "No notifications yet" centered `font-cond text-[12px] text-muted` at 80px height
- Footer link: "Notification Settings →" — opens `NotificationSettingsPanel` (sets active settings tab)
- Close: click outside or press Escape — follow existing TopBar dropdown `mousedown` handler pattern

Source: CONTEXT.md D-04 + `TopBar.tsx` dropdown pattern

### NotificationSettingsPanel
- Accessed via: Settings tab (new sub-section) or via dropdown "Notification Settings" link
- Layout: follows `EventSetupTab` settings section pattern — full-width panel with `SectionHeader` dividers and `Card` containers
- Sections by alert type (D-02): Weather Alerts, Schedule Changes, Admin Alerts, Registration Updates
- Role filtering (D-03): render only relevant sections per role — admins see all 4, coaches see Weather + Schedule, program leaders see Registration + Schedule
- Each section contains: alert type name (`font-cond text-[13px] font-black text-white`), description (`text-[12px] text-muted`), email toggle row, push toggle row
- Toggle: custom toggle switch — `w-10 h-5 rounded-full` background (`bg-navy` when on, `bg-[#1a2d50]` when off), `w-4 h-4 rounded-full bg-white` thumb, `transition-transform duration-150`
- Save: `Btn` primary variant, "Save Preferences" label, positioned at panel bottom-right
- Success feedback: `toast.success('Preferences saved')` — matches existing toast pattern

Source: CONTEXT.md D-01, D-02, D-03 + `EventSetupTab.tsx` patterns + CLAUDE.md component patterns

### Push Permission Prompt (D-09)
- Trigger: after first meaningful action (schedule view loaded, command center opened) — NOT on first page load
- Component: `Modal` (existing component) — title: "Get Event Alerts", body: "Stay updated on weather delays, schedule changes, and game day alerts for [Event Name].", footer: `Btn` primary "Enable Notifications" + `Btn` ghost "Not Now"
- Context message must include the event name (from `state.event?.name`)
- Do not re-prompt if permission previously denied or granted (check `Notification.permission` before showing)
- aria-describedby on modal body paragraph

Source: CONTEXT.md D-09

### Push Notification Collapse (D-11)
- 3+ push notifications within 60 seconds collapse to summary
- Summary title: "New Alerts — [Event Name]"
- Summary body: "[N] new alerts — Tap to view"
- Click opens app to notification dropdown
- Implementation: service worker `push` event handler logic — not a UI component

Source: CONTEXT.md D-11

## Animation & Transition Contract

| Interaction | Duration | Property | Easing |
|------------|----------|----------|--------|
| Bell hover color | 150ms | color | ease |
| Dropdown open/close | 150ms | opacity + transform (translateY -4px → 0) | ease-out |
| Toggle switch thumb | 150ms | transform (translateX) | ease |

## Accessibility Requirements

- Bell button: `aria-label` with unread count, `aria-haspopup="true"`, `aria-expanded={isOpen}`
- Dropdown: `role="menu"`, items use `role="menuitem"`
- Preferences toggles: `role="switch"`, `aria-checked={enabled}`, `aria-label="{alert type} {channel} notifications"`
- Push modal: `aria-describedby` on body paragraph
- Focus management: when dropdown opens, focus first item; when it closes, return focus to bell button
- Keyboard: Escape closes dropdown, arrow keys navigate items, Enter/Space activates
- Color: unread indicator uses both color AND badge count number — color is not the sole indicator

Source: ui-ux-pro-max accessibility rules (color-contrast, aria-labels, keyboard-nav, focus-states)
