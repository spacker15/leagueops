**Fix 4: Realtime useEffect -- REVIEW FIX #3 (MEDIUM): Use currentDateRef to prevent reconnect storm**

The cross-AI review identified that using `[eventId, currentDate]` as the realtime effect dep array causes channel teardown on every date tab switch -- a reconnect storm on tournament day. Instead, use a `useRef` for currentDate so the games callback reads the ref value without the ref being a dependency.

First, add a ref and sync effect near the top of AppProvider (before the realtime useEffect):

```typescript
const currentDateRef = useRef(currentDate)
useEffect(() => {
  currentDateRef.current = currentDate
}, [currentDate])
```

Then update the realtime subscription useEffect to depend ONLY on `[eventId]`:

```typescript
useEffect(() => {
  if (!eventId) return // D-01: null guard
  const sb = createClient()
  const filter = `event_id=eq.${eventId}`
  const sub = sb
    .channel('leagueops-realtime')
    .on(
      'postgres_changes',
      { event: '*', schema: 'public', table: 'ops_log', filter },
      (payload) => {
        dispatch({ type: 'ADD_OPS_LOG', payload: payload.new as OpsLog })
      }
    )
    .on('postgres_changes', { event: '*', schema: 'public', table: 'incidents', filter }, () => {
      db.getIncidents(eventId).then((d) => dispatch({ type: 'SET_INCIDENTS', payload: d }))
    })
    .on('postgres_changes', { event: '*', schema: 'public', table: 'games', filter }, () => {
      const cd = currentDateRef.current // read from ref, not state
      if (cd) {
        db.getGamesByDate(eventId, cd.id).then((d) => dispatch({ type: 'SET_GAMES', payload: d }))
      }
    })
    .on(
      'postgres_changes',
      { event: '*', schema: 'public', table: 'medical_incidents', filter },
      () => {
        db.getMedicalIncidents(eventId).then((d) => dispatch({ type: 'SET_MEDICAL', payload: d }))
      }
    )
    .subscribe()
  return () => {
    sb.removeChannel(sub)
  }
}, [eventId]) // ONLY eventId -- currentDate read from ref. Addresses review concern #3.
```

Key changes vs original plan: dep array is `[eventId]` only (not `[eventId, currentDate]`), games callback reads `currentDateRef.current` instead of `currentDate`. This prevents channel teardown/recreation when switching date tabs.
