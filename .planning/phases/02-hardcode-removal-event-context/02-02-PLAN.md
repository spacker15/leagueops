**REVIEW FIX #4 (MEDIUM): Wire invalidateRulesCache after mutations.**

The file already imports `invalidateRulesCache` from `@/lib/engines/rules` but never calls it. After EVERY successful POST (insert) and PATCH (update) Supabase call in this route, add:

```typescript
invalidateRulesCache(Number(event_id))
```

Place it after the `.insert()` or `.update()` call succeeds (after the `if (error)` check), before returning the success response. This ensures the in-memory rules cache for this event is evicted so subsequent reads pick up the mutation. This addresses the cross-AI review concern that the 30-second TTL cache could serve stale rules after an admin edit.
