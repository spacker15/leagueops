import { render, act, waitFor } from '@testing-library/react'

**Test cases to write:**

**SEC-04 Tests:**

1. **"loadAll re-fires when eventId prop changes"**
   - Render `<AppProvider eventId={1}>` with a child that reads `useApp()`
   - Wait for initial loadAll to complete (mock resolves)
   - Re-render with `eventId={2}`
   - Assert that `db.getEvent` (or the underlying Supabase `.from('events')` call) was called again with the new eventId
   - Verify: the mock's call count for data-fetching functions increases after re-render

2. **"loadAll does not fire when eventId is undefined"**
   - Render `<AppProvider eventId={undefined as any}>` (simulating loading state)
   - Assert that no data-fetching calls were made
   - This validates the D-01 null guard

**SEC-05 Tests:**

3. **"realtime subscription includes event_id filter"**
   - Render `<AppProvider eventId={5}>`
   - Wait for subscription setup
   - Assert that the mock channel's `.on()` was called with config objects containing `filter: 'event_id=eq.5'`
   - Check all 4 tables: ops_log, incidents, games, medical_incidents

4. **"realtime channel is torn down and recreated when eventId changes"**
   - Render `<AppProvider eventId={1}>`
   - Wait for subscription
   - Record the removeChannel mock
   - Re-render with `eventId={2}`
   - Assert: `removeChannel` was called (teardown of old channel)
   - Assert: new `.channel()` call was made
   - Assert: new `.on()` calls include `filter: 'event_id=eq.2'`

5. **"realtime subscription does not start when eventId is undefined"**
   - Render `<AppProvider eventId={undefined as any}>`
   - Assert: no `.channel()` or `.subscribe()` calls were made
