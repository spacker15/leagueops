<feature>
  <name>Slot Suggestion Engine</name>
  <files>lib/engines/schedule-change.ts, __tests__/lib/engines/schedule-change.test.ts</files>
  <behavior>
    - generateSlotSuggestions returns up to 5 SlotSuggestion objects sorted by score descending
    - Returns empty array when all candidate slots have field or team conflicts
    - Filters out slots where the field already has a game overlapping the candidate time window
    - Filters out slots where either team (home or away) has a game overlapping the candidate time window
    - Scores same-day slots +100 over the original game date
    - Scores adjacent-day (1 day) slots +50, 2-day slots +25
    - Scores +20 for each team that has the candidate event_date in their available_date_ids
    - Sets homeTeamAvailable=true when home team has no conflict AND has the date in availability
    - Sets awayTeamAvailable=true when away team has no conflict AND has the date in availability
    - Skips past event dates (only future dates are candidates)
    - Skips the game's current slot (same field + same time)
    - Handles edge case: game with no other games in event returns all candidate slots
  </behavior>

**Exported function:**
```typescript
export function generateSlotSuggestions(params: {
  game: Game
  allGames: Game[]
  fields: Field[]
  eventDates: EventDate[]
  teamAvailability: Record<number, number[]>  // teamId -> available event_date_ids
  gameDurationMin: number
}): SlotSuggestion[]
```

**Internal helpers:**
- `overlaps(aStart: Date, bStart: Date, durationMin: number): boolean` — returns true if two time windows of `durationMin` length overlap
- `getCandidateTimes(eventDate: EventDate, gameDurationMin: number): string[]` — returns hourly time strings from 08:00 to 18:00 stepping by gameDurationMin (minimum 60 min steps)

**Algorithm per RESEARCH.md Pattern 5:**
1. Iterate eventDates x fields x candidateTimes
2. Skip past dates (isAfter check against now)
3. Skip same slot as current game
4. Check field conflict: any existing game on same field overlaps candidate
5. Check team conflicts: home team or away team has overlapping game
6. If field conflict, skip entirely (hard filter)
7. If team conflict, still include but mark homeTeamAvailable/awayTeamAvailable = false
8. Score: same day +100, 1 day diff +50, 2 day diff +25, per-team availability +20 each
9. Sort descending by score, return top 5

**IMPORTANT:** The engine does NOT import date-fns for the overlap math — use raw Date arithmetic (getTime() + durationMin * 60000). Only use date-fns for isSameDay, isAfter, differenceInCalendarDays for date-level comparisons.
