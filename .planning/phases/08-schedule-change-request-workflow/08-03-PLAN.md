**1. `app/api/schedule-change-requests/route.ts`** — GET + POST

GET handler:
- Auth check via `supabase.auth.getUser()`
- Read `event_id` from URL search params: `const eventId = Number(req.nextUrl.searchParams.get('event_id'))`
- If no eventId or isNaN: return 400 `{ error: 'event_id required' }`
- Query `schedule_change_requests` with select `*, team:teams(id, name), games:schedule_change_request_games(*, game:games(*))` filtered by `.eq('event_id', eventId)`, ordered by `created_at desc`
- Return 200 with `{ data }` where data is the array

POST handler:
- Auth check via `supabase.auth.getUser()`
- Parse JSON body, validate with `createScheduleChangeRequestSchema`
- Read `event_id` from search params (same pattern)
- Determine user role: query `user_roles` where `user_id = user.id` and `event_id = eventId`. Check role is 'coach' or 'program_leader'. If not, return 403 `{ error: 'Only coaches and program leaders can submit requests' }`
- Verify all game_ids belong to the requesting team: query `games` where `id IN game_ids` and `(home_team_id = team_id OR away_team_id = team_id)` and `event_id = eventId`. If count !== game_ids.length, return 400 `{ error: 'Invalid game selection' }`
- Insert request row into `schedule_change_requests` with fields: `event_id`, `submitted_by: user.id`, `submitted_by_role: userRole.role`, `team_id`, `request_type`, `reason_category`, `reason_details`, `status: 'pending'`
