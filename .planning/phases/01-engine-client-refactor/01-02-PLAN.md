# Plan B1: New API Routes for CommandCenter

## Goal

Create 3 new server-side API route handlers that wrap the unified engine functions, so `CommandCenter.tsx` can call them via `fetch` instead of importing engine functions directly from a client component.

## Review Feedback Addressed

- **HIGH — Plan A + B wave 1 parallelism is fragile**: This plan (B1) is the wave 1 portion split from the original Plan B. It contains only the new route files (no deps on Plan A signatures since these files are created fresh). Plan B2 is the wave 2 portion that updates existing routes and depends on Plan A.
- **MEDIUM — New API routes lack error handling specification**: All 3 routes use the `{ error: string }` pattern with appropriate HTTP status codes, matching existing routes in the codebase (`app/api/games/route.ts`, `app/api/fields/route.ts`, etc.). Specifically: `400` for missing/invalid request body fields, `500` for engine errors caught in `try/catch`.
