```typescript
const coachConflictPairs = await getConflictingTeamPairs(eventId, sb)
```

3. Create helper function (can be inside generateSchedule or module-level):

```typescript
function teamsShareCoach(teamA: number, teamB: number): boolean {
  const key = `${Math.min(teamA, teamB)}-${Math.max(teamA, teamB)}`
  return coachConflictPairs.has(key)
}
```

4. In the slot assignment inner loop — where the engine evaluates whether a matchup can be placed into a specific time slot — add a check:

```typescript
// Check if any game already placed in this same slot involves a team that shares a coach with current matchup
const coachConflictInSlot = gamesInThisSlot.some(
  (placed) =>
    teamsShareCoach(placed.home_team_id, matchup.home) ||
    teamsShareCoach(placed.home_team_id, matchup.away) ||
    teamsShareCoach(placed.away_team_id, matchup.home) ||
    teamsShareCoach(placed.away_team_id, matchup.away)
)
if (coachConflictInSlot) continue // skip this slot for this matchup
```

CRITICAL per RESEARCH Pitfall 1: This is a SLOT-LEVEL constraint, NOT a matchup-level filter. Two teams sharing a coach CAN still play each other — they just cannot play simultaneously with OTHER teams sharing that coach. Do NOT add to the `filteredMatchups` step.

5. If `coachConflictPairs` is empty (no conflicts), the check is a no-op — no performance impact.
