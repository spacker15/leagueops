**Part A: Create `components/events/MultiDatePicker.tsx`**

A new client component that renders a month-view calendar grid between event start_date and end_date. Admin clicks day cells to toggle selection. Uses `date-fns` for date arithmetic.

Props:

```typescript
interface MultiDatePickerProps {
  startDate: string // event start_date ISO string
  endDate: string // event end_date ISO string
  selectedDates: string[] // array of ISO date strings currently selected
  onToggleDate: (date: string) => void // called when a day cell is clicked
}
```

Implementation details:

- Use `eachDayOfInterval`, `parseISO`, `format`, `startOfMonth`, `endOfMonth`, `startOfWeek`, `endOfWeek`, `isSameMonth`, `isSameDay` from `date-fns`
- Generate calendar grid: for each month in range, render a 7-column grid (Sun-Sat headers)
- Day cell: `px-3 py-3` (12px padding per UI-SPEC spacing exception), clickable
- Selected day: `bg-[#0B3D91] text-white rounded-lg` (navy accent per UI-SPEC)
- Unselected day (in range): `text-white hover:bg-[#0a1a3a] rounded-lg cursor-pointer`
- Out-of-range day (before start_date or after end_date): `text-[#2a3a5a] cursor-not-allowed` (dimmer than muted)
- Day name headers: `text-[12px] text-[#5a6e9a] font-cond font-black tracking-[.12em] uppercase`
- Month header: `text-[18px] font-cond font-black text-white` with left/right arrows if multi-month
- Must support multi-month view (render each month as a separate grid when range spans months)
- `'use client'` directive at top

**Part B: Extend EventSetupTab General tab**

In the General tab section of EventSetupTab, add THREE new sections below existing event date fields:

**Section 1: Schedule Dates (per D-11)**

- SectionHeader "SCHEDULE DATES"
- Render MultiDatePicker between event start_date and end_date
- On date toggle: POST to insert or DELETE from `event_dates` table
  - Insert: `{ event_id, date: isoDate, label: format(date, 'EEEE'), day_number: N }` where N is sequential
  - Delete: delete by event_id + date
- Load existing event_dates on mount via `sb.from('event_dates').select('*').eq('event_id', event.id)`
- `toast.success('Schedule dates saved')` on successful save

**Section 2: Registration Window (per D-13, D-14, D-15)**

- SectionHeader "REGISTRATION WINDOW"
- Two datetime-local inputs:
  - Label "REGISTRATION OPENS" — `<input type="datetime-local">` bound to `registration_opens_at`
  - Label "REGISTRATION CLOSES" — `<input type="datetime-local">` bound to `registration_closes_at`
  - Both use `inp` class variable (dark theme input styling from EventSetupTab)
  - Both use `lbl` class variable for labels
- Registration toggle with three visual states per UI-SPEC:
  - When no manual override: toggle shows "Auto" in navy if dates are set, or gray "OFF" if no dates
  - When manually ON: toggle green with label "Override: Open"
  - When manually OFF: toggle red with label "Override: Closed"
  - Toggle is a clickable div with `role="switch" aria-checked={value}` per UI-SPEC accessibility
  - Toggle cycles: auto → manual open → manual closed → auto
  - Internal state maps to `registration_open`: `true` = manual open, `false` = manual closed, `null` (or remove field) = auto/date-based
- Btn variant="primary" "SAVE REGISTRATION SETTINGS" — calls saveSettings with the new fields
- Spinner during save, `toast.success('Registration settings saved')` on success

**Section 3: Update EventData and saveSettings**

- Add `registration_opens_at`, `registration_closes_at`, `registration_open` to the `EventData` interface and `DEFAULT_EVENT` object in EventSetupTab
- Update `loadEvent` to fetch these new columns
- Update `saveSettings` to PATCH these fields to `/api/events/[id]` (or direct Supabase update, matching existing pattern)
